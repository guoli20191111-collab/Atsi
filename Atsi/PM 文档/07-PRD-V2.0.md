## 二、项目概述

### 2.1 项目背景

人格测试赛道经历了三阶段(详见 02-MRD):专业测试 → 泛专业 → 社交内容化。当下 Z 世代对 MBTI 已**审美疲劳**,但又对"娱乐壳测试"觉得不够分。市场上**专业感和年轻化调性的交集象限几乎空白**,这是 ATSI 的机会。

### 2.2 项目目标

**核心目标:** 打造一款"接地气、有共鸣、网红口语风"的人格测试产品,在 25-35 岁人群中建立"MBTI 反义词"的心智。

**V2 阶段目标:**
1. 把 MVP 跑通的产品形态,升级为"可被传播 + 可被衡量"的产品
2. 验证核心假设:用户是否真的愿意分享 / 复测
3. 沉淀可被 V3+ 复用的数据基础设施

### 2.3 项目范围

| 在范围内 | 不在范围内 |
|---|---|
| 落地页 / 测试页 / 结果页 体验优化 | 用户账号/登录体系 |
| 60 题 → 30 人格 的打分逻辑代码化 | 付费深度报告 |
| 结果页分享卡片可视化 | 移动端原生 App |
| 基础埋点 + 数据回收 | 多语言版本 |
| 30 人格独立详情页(人格百科) | 真人客服 / 用户社区 |
| 历史结果本地存储 + 对比 | 后台管理系统(题库管理走文档) |
| 简版 10 题 / 深度版 60 题(P2)| 视频/音频版本 |

#### 4.3.1 抽题算法(方案 B)规范

> 当前为文档定义,V2 必须代码化

```
输入:60 题题库,每题有 [维度编号 1-15] + [风格映射 ABCD]
输出:30 题列表,满足以下约束:

约束 1:维度均衡
- 每个维度至少抽 2 题,最多抽 2 题(15 维度 × 2 题 = 30 题)

约束 2:风格分布约束
- 30 题中,A(躺平)/ B(自律)/ C(外向)/ D(随性)四种风格作为
  "正确答案"出现的次数,差值不超过 ±2

约束 3:锚点题占比
- 4 选项题(含 D"自己也不知道")占 ~10% = 3 道左右

约束 4:随机性
- 在满足以上约束下,随机抽选,保证用户每次测试题目不完全相同
- 同用户 7 日内复测题目重叠不超过 60%(本地 localStorage 记录)
```

### 4.4 打分引擎(V2 P0,核心新增)

> V1 阶段算法只在文档里,V2 必须代码化为可执行函数

#### 4.4.1 打分流程

```
用户作答 30 题
        ↓
[Step 1: 风格累计]
每题选项 → 对应风格(A/B/C/D)累加 +1
得到:风格向量 [A: 12, B: 8, C: 7, D: 3]
        ↓
[Step 2: 维度推算]
对每个维度(15 个):
  取该维度下的 2 道题的作答
  根据答案映射到该维度的 H(高)/M(中)/L(低)
得到:维度向量 [维度1: H, 维度2: M, 维度3: H, ...]
        ↓
[Step 3: 人格匹配]
30 个人格,每个都有自己的 15 维度向量
计算用户向量与每个人格向量的"距离"(海明距离或加权)
取距离最小的人格 = 主人格
取距离次小的人格 = 副人格(V2 P2 展示)
        ↓
输出:主人格 + 副人格 + 匹配度百分比
```

#### 4.4.2 匹配度计算

```
匹配度 = (1 - 距离 / 最大可能距离) × 100%
显示:88%-99% 范围(永远不显示 100%,留余地)
```

#### 4.4.3 算法验收标准

- 30 个人格在 1000 次随机模拟测试下,被匹配的频次方差 ≤ 阈值(避免某个人格永远不会被匹配)
- 用户主观打分"匹配度感知" ≥ 4.0/5.0(V2 上线后 4 周收集)

## 五、测试题目规范(承袭 V1,补充细则)

### 5.1 题目设计原则

- **题干**:围绕日常生活场景(休息、决策、社交、情绪),避免抽象晦涩
- **选项**:网红口语风,A/B/C 对应"躺平摆烂 / 积极自律 / 随性松弛",约 10% 题目加 D"自己也不知道"
- **长度**:选项 1-2 句话,有画面感(如"一觉睡到自然醒,醒后刷手机,不轻易下床")

### 5.2 禁忌

- ❌ 低俗网络梗 / 不雅表述
- ❌ 文艺鸡汤体 / 书面化表达
- ❌ 选项模糊 / 逻辑矛盾
- ❌ 心理学专业术语

### 5.3 题目质量验收

新增题目需通过:
1. 风格 review:是否符合"网红口语 + 傻萌智慧"
2. 维度归属:是否能清晰映射到 15 维度之一
3. 风格映射:选项 A/B/C(/D)是否清晰对应 4 风格

## 八、异常 / 边界 case

| 场景 | 处理方式 |
|---|---|
| 用户测到一半关闭页面 | localStorage 暂存,30 分钟内回来可继续,超时则丢弃 |
| 用户网络中断 | 答题数据本地暂存,网络恢复后上报 |
| 抽题算法找不到满足约束的题目组合 | 优先满足维度均衡,放宽风格分布约束 |
| 多个人格匹配度相同 | 按预设优先级(派系平衡 → 用户历史结果差异化)选择 |
| 用户多次刷新结果页 | 同次答题 session 内,结果保持不变(避免每次刷新都重新随机) |
| 分享卡片下载失败 | 提供"长按截图"备选方案 |
| 浏览器不支持 canvas | 降级为纯 HTML 截图引导 |

## 九、验收标准

### 9.1 功能验收

- [ ] 用户从落地页到看到结果页,全流程顺畅(无报错、无白屏)
- [ ] 30 题抽题严格满足方案 B 约束(自动化测试 1000 次验证)
- [ ] 打分算法对 30 个人格的匹配频次方差 ≤ 设定阈值
- [ ] 分享卡片在小红书/朋友圈/抖音/微博四种尺寸下显示正确
- [ ] 历史结果本地存储读写正常
- [ ] 人格百科 30 个独立页面全部可访问
- [ ] 全部埋点上报正确

### 9.2 体验验收

- [ ] 5 名陌生用户完测,无中途放弃
- [ ] 5 名用户中至少 4 名表示"会分享给朋友"

- [ ] 分享卡片在小红书发布后,视觉质感不低于现有头部测试 KOL 的卡片

### 9.3 数据验收(上线后 4 周)

- [ ] 完测率 ≥ 60%
- [ ] 分享率 ≥ 25%
- [ ] 匹配度感知评分 ≥ 4.0/5.0
- [ ] 30 人格分布均匀度(频次方差)≤ 设定阈值
- [ ] 7 日复测率 ≥ 15%
